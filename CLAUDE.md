# VibeCode Workshop

A 2-session Claude Code onboarding workshop. Designed for non-engineers (no
CLI, git, or AI tooling experience required). Runnable as a facilitated group
session or as a self-paced solo tutorial.

## Purpose

Take participants from a fresh setup to: a working Claude Code workflow, a
real first project on GitHub, an AI-generated Figma design, a built prototype,
and hands-on experience collaborating via git branches.

## Structure

- `01-pre-work/` — participant pre-work for Session 1 (with facilitator prep at top)
- `02-session-1/` — Session 1 walkthrough (environment + CC fundamentals + first project)
- `03-pre-work-session-2/` — participant pre-work for Session 2 (with facilitator prep at top)
- `04-session-2/` — Session 2 walkthrough (design, prototype, GitHub collaboration)
- `ideas/` — future workshop content (e.g., Figma plugin build)
- `feedback/` — notes from real workshop runs

## Guidelines when working on this project

- Session guides and pre-work are **participant-facing tutorials**, not
  facilitator scripts. Direct second-person voice.
- Facilitator-prep checklists live at the top of each pre-work file under a
  "For facilitators (or solo learners)" section.
- Inside session guides, use `> **For facilitators:**` callouts only for
  pointers — don't duplicate prep checklists; reference the pre-work file.
- Assume zero CLI, git, or coding experience.
- Commands shown must be copy-paste ready and tested.
- Default to **CC-first** for actions; show terminal commands as `> Tip:`
  blocks underneath when manual fallback is useful.
- Keep session timing realistic — buffer for questions and small failures.
